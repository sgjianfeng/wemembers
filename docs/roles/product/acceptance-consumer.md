## 1. 消费者心智模型（产品定义）

```
发现门店 → 关注/进店 → 买券或抽奖 → 到店核销 → 再来 / 再发现
     ↑____________ 最近去过 / 收藏 _____________|
```

| 概念 | 产品含义 | 数据载体（现状） | UI 现状 |
|------|----------|------------------|---------|
| **用户账号** | 手机号消费者 | `User.role=customer` | 注册/登录有，但不稳 |
| **门店** | 可到店消费的地点 | `Store` | 商家侧强；消费者发现弱 |
| **关注门店** | 用户主动收藏，方便再来 | `Membership.isFavorite` | **无消费者 UI** |
| **最近去过** | 核销/访问过的店 | `RedemptionLog` / 访问日志 | **无** |
| **查找门店** | 按名/区域/品类搜 | 无专用索引 API | **无** |
| **买券/抽奖** | 活动页 `/voucher/{slug}` | Campaign + Voucher | 有链接即可买（Test） |
| **我的余额/券** | 已购权益 | Voucher / CustomerCoupon | `/balance` `/wallet` 部分有 |

## 2. 关键路径验收（P0 — 必须先过）

环境：生产 `https://wemembers.store` · **无痕** · 消费者身份

| # | 步骤 | 期望 | 状态 | 备注 |
|---|------|------|------|------|
| C-01 | 打开首页「我是消费者」 | 无 key 闪烁；无商家试点链接；顶栏逻辑对 | 🔄 | i18n 修复中 |
| C-02 | 点免费注册 → 选客户 | 文案正常中文，非 `auth.register.*` | 🔄 | 同 i18n |
| C-03 | 填手机号发验证码 | 成功提示；能收到 SMS 或明确失败原因 | ❌ | Vonage 422 / 号码格式 |
| C-04 | 验证码完成注册 | 进 `/home`，会话为 customer | ❌ | 依赖 C-03 |
| C-05 | 登录页 tab=customer | 密码/验证码可登已有用户 | 🔄 | 需已有账号 |
| C-06 | 首页/登录顶栏身份 | 消费者不显示商家名 | ✅ | 已按 Tab 区分 |
| C-07 | 打开已知活动链接买券 | Checkout Test 卡可付 | ✅ | 专链，非首页 |
| C-08 | 购后余额可见 | `/balance` 有记录 | 🔄 | 需买通后验 |

**P0 退出标准：** C-01～C-05、C-07 全绿。  
**当前：未退出。**

## 3. 门店关系（P1 — 产品应有，现未齐）

| # | 能力 | 用户故事 | 状态 | 建议排期 |
|---|------|----------|------|----------|
| S-01 | **查找门店** | 作为用户，我能按名字找到 Meow BBQ | ❌ 无 | I1 后半 |
| S-02 | **门店主页** | 看到店信息、活动、导航去买券 | 🔄 有 `/shop` `/store` 不完整 | 打磨 |
| S-03 | **关注门店** | 点关注后在「我的门店」看到 | ❌ 仅 schema `isFavorite` | I1 |
| S-04 | **最近去过** | 按核销时间列出去过的店 | ❌ | I1 |
| S-05 | **从首页进店** | 消费者首页有发现区，不是空白游戏化堆砌 | ❌ 首页偏营销 | 重做 IA |

**原则：** 没有 S-01～S-04，**不要在首页假装有「周边门店生态」**。

## 4. 首页 IA（产品要求）

| 区块 | 消费者 Tab 应有 | 禁止 |
|------|-----------------|------|
| 顶栏 | 登录 / 用户名（仅 customer 会话） | 管理后台、商家店名冒充用户 |
| Hero | 通用价值主张 + 注册/登录 | 某店 S$10 / 抽奖三档硬编码 |
| 中部 | 怎么玩（通用）+ 可选「热门活动」数据驱动 | 假数据门店列表 |
| 下部 | 去登录后的入口说明 | 未实现能力的入口按钮 |

商家 Tab：注册企业 / 已登录进后台。与消费者 **会话隔离叙事** 一致。
