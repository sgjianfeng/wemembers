# 产品决策 log

## 2026-07-19 · 试点产品边界（大股东确认）

### D-P1-1 代金券 vs 抽奖：各是各的

- **决定**：
  1. **S$10 代金券** = 纯「折扣代金券」活动（`voucher_discount` / `voucher_sale`），**无抽奖**。
  2. **抽奖活动** = 独立「梦想大奖池」（`draw_standard` / `lucky_draw_v2`），购券面额为 **三档：S$50 / S$100 / S$200**。
  3. 两条活动 **分开创建、分开链接、分开购买、分开对账**。
- **不做什么**：不为了试点把 S$10 塞进抽奖三档；不把三档抽奖拆成「3 张纸质券」语义。
- **账目验收**：大股东各买 **1 笔 S$10 代金** + **1 笔抽奖任一档**，核对实付 / 余额 / 核销分账 / 奖池。

### D-P1-2 抽奖「3 档券」定义

- **决定**：**3 档 = 三种购券面额**（50 / 100 / 200），不是 3 个大奖名称。  
  大奖展示（iPad / iPhone / BYD 等）是奖池 UI，与档位无关。

### D-P0-1 企业 vs 门店信息架构

- **决定**：企业账号进 **公司 Dashboard**；具体店务进入 `/business/stores/[id]`；**无全局「当前门店」切换**。店员账号绑定店，登录即本店。  
- **原因**：符合真实连锁组织；减少错误核销记错店。  
- **复盘**：试点店员是否需要更短路径（桌面书签到 scan）。

### D-P0-2 I0 范围

- **决定**：只保证 **注册 → 设置 → 门店 → 码/核销选店**；发券/抽奖「能点开不崩」即可，不深打磨。  
- **原因**：市场演示卡在 onboarding 比卡在奖池配置更致命。  

### D-P0-3 完成定义

- **决定**：完成 = `acceptance.md` 全绿，不是「合并了很多 PR」。
