# 试点演示脚本（约 15 分钟）

## 开场（1 分钟）

「WeMembers 帮门店做两件事：发代金券拉客，以及买券抽奖促转化。  
**没有月费**，企业开账号、门店贴码，顾客扫码，到店核销。」

## 演示顺序（10 分钟）

| 步 | 你操作 | 一句话 |
|----|--------|--------|
| 1 | 打开注册/已准备好的企业号 | 公司主体（UEN）注册，不是先建一大堆店员 |
| 2 | 企业设置：公司名 + 英文网址标识 | 品牌与链接固定 |
| 3 | 添加 1–2 个门店 | 连锁可以多家，核销记在具体店 |
| 4 | 进门店：二维码 / 顾客页链接 | 打印贴收银台 |
| 5 | 核销入口：选店或本店核销 | 店员只看到本店（后续试点可开店员号） |
| 6 | （可选）发券或抽奖活动入口 | 先证明核销闭环，再谈大奖 |

## 异议处理

| 商家说 | 回应 |
|--------|------|
| 和 Grab 券有什么不同？ | 你们自己的品牌资产与会员；跨店结算与抽奖可配置 |
| 抽奖会不会亏？ | 预算比例可配；I0/I1 可先只用代金券 |
| 要不要很贵的系统费？ | 零月费起步；平台从服务费模型走；自营店先跑通 |
| 员工会用吗？ | 店员端只核销，手机号 + 密码登录，步骤比总部少 |

## 收尾邀约（2 分钟）

「给我们 2 周试点：帮你们开好企业 + 1 家门店 + 一张活动/券。  
你们只需要贴码、试核销。结束时一起看数据决定去留。」

### 登录方式建议（产品现状，I0 不新做 PIN）

当前系统 **没有独立 PIN 登录**，只有：

| 角色 | 账号标识 | 推荐登录 | 说明 |
|------|----------|----------|------|
| **企业主（你）** | **邮箱** | 邮箱 + **密码** | 注册企业必须邮箱；日常后台、发券、建活动用这个 |
| **店长 / 店员** | **手机号** | 手机 + **密码**（主） / 短信验证码（备用） | API `POST .../stores/[id]/staff` 只接受 `phone` 邀请；登录后进本店核销 |
| 顾客 | 手机为主 | 密码或验证码 | 与试点后台无关 |

**推荐组合（Meow BBQ）：**

1. **你**：`meow.jianfeng@gmail.com` + 强密码（企业号，永不给前台共用）  
2. **店长/员工**：各自 **手机号** + 简单密码（≥6 位）；忘记密码用短信验证码登录后再设密码  
3. **不要做「全店共用一个邮箱」**；也 **不必等 PIN 功能**——收银场景手机 + 密码已够用  

若以后要「4 位数 PIN 快速核销」，进 backlog（P1/P2），不进 I0。
