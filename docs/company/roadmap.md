# Roadmap（阶段制，非功能清单）

> 与 `NORTH_STAR.md` 绑定。功能细节在 `docs/roles/product/backlog.md`。

| 阶段 | 名称 | 退出标准（Exit） | 状态 |
|------|------|------------------|------|
| **A** | 可试点 | 企业主路径稳定；可给商家做 15 分钟演示；无登录/设置/加店阻塞 | 🔄 I0 |
| **B** | 试点运营 | ≥3 店完成 onboarding；有真实核销；有问题清单与修复节奏 | 未开始 |
| **C** | 可复制获客 | 一种获客方式可重复；单位经济可讲清 | 未开始 |
| **D** | 规模化 | 自动化结算/风控/支持；多垂直复制 | 未开始 |

## 近期迭代意图

| 迭代 | 窗口（建议） | 主题 | 主导角色 |
|------|--------------|------|----------|
| **I0** | 2026-07-18 → 07-31 | 打底：主路径稳定、治理就位、试点话术 | 工程 + 产品 |
| **I1** | 08-01 → 08-14 | 试点包：onboarding 清单、演示脚本、核销闭环 | 产品 + 市场 |
| **I2** | 08-15 → 08-28 | 第一批试点反馈 → 修最痛的 3 点 | 全员 |
| **I3+** | 视试点 | 获客实验或抽奖/结算加深 | 市场定 |

日期可按大股东资源平移，**主题不要并行超过一个主路径**。
