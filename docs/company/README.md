# WeMembers 公司层治理（轻量）

本目录是**唯一的事实源**（目标、阶段、大股东决策）。  
角色视角见 `docs/roles/`。代码与功能实现仍在仓库主线。

## 角色与目录

| 角色 | 目录 | 职责 |
|------|------|------|
| 大股东（你） | `docs/company/decisions/` | 资源、客群、定价、渠道拍板 |
| 市场总监（30% 视角） | `docs/roles/market/` | 获客叙事、实验、竞争时机 |
| 产品 + 运营 | `docs/roles/product/` | 路径、优先级、验收、运营节奏 |
| 架构 + 开发 | `docs/roles/eng/` | 实现切片、质量、技术债 |

## 节奏（默认）

- **迭代长度：14 天**
- **D0**：市场 → 产品 → 工程 各半页 → 大股东只答 4 问（见 `NORTH_STAR.md`）
- **D1–D10**：工程交付；产品中段验收
- **D11–D12**：Demo + 验收勾选
- **D13–D14**：三角色复盘 → 更新 roadmap → 下轮假设

## 当前迭代

见 [`iterations/I00.md`](./iterations/I00.md)（Iteration 0：打底与试点准备）。

## 原则

1. **一个目标，三个视角** — 禁止各角色自写另一套成功标准  
2. **每轮 1 条主路径 + 最多 1 个增长杠杆**  
3. **决策半页内** — 背景 / 选项 / 决定 / 复盘日  
4. **冲突升级**：市场 vs 产品 → 大股东；产品 vs 工程 → 砍范围不砍底线
