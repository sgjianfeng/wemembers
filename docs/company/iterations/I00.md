## 自上而下决策摘要

### 市场（详见 `docs/roles/market/`）

- **叙事**：零月费起步；企业管活动，门店贴码核销；买券抽奖是亮点不是唯一入口  
- **本迭代获客**：不做投放；只准备「试点邀约话术 + 演示脚本」  
- **成功**：能向 3 家目标商家完整演示而不因产品缺陷中断  

### 产品（详见 `docs/roles/product/`）

- **主路径（P0）**  
  1. 注册企业（邮箱 + UEN + 密码）  
  2. 企业设置（公司名、英文 slug）  
  3. 添加 ≥1 门店  
  4. 打开门店页 / 二维码  
  5. 企业核销时选店（或从门店进入核销）  
  6. （可选）创建一张券或看到活动入口  
- **本迭代不做**：店员完整 HR 流程、跨店复杂结算 UI 大改、消费者大改版  

### 工程（详见 `docs/roles/eng/`）

- 会话失效（清库后 JWT）不再死循环  
- 企业设置页稳定  
- 公司后台无「当前门店」全局切换；进店再核销  
- 关键路径冒烟可脚本/手工复现  
- 已知阻塞：会员列表 onChange SSR 问题已修；继续扫其他 Server Component 事件处理器  

## Definition of Done（I0）

- [ ] 空库或新浏览器：注册企业 → 设置 → 加 2 店 → 进店 → 核销选店，全流程无 500  
- [ ] 旧 cookie / 失效用户：自动清会话到登录，无 replaceState 风暴  
- [ ] 落地页 hydration 无 roleView 类报错（硬刷新验证）  
- [ ] `docs/roles/product/acceptance.md` 验收项全部勾选  
- [ ] 市场：`docs/roles/market/pitch-pilot.md` 演示脚本可用  
- [x] 大股东完成 `decisions/2026-07-18-start-governance-i0.md` 拍板栏
